# ARGUS

ARGUS is a multi-agent quantitative research terminal built as a learning-first,
production-minded full-stack system.

The project combines:

- FastAPI backend services
- React and TypeScript frontend workflows
- PostgreSQL-backed persistence
- Quantitative analytics
- AI-assisted research workflows
- Agent orchestration
- Retrieval-augmented generation
- Real-time terminal-style dashboards

## Current Phase

Phase 1: Foundations

Before writing significant application code, this phase teaches the core ideas
that every later system depends on:

- HTTP
- REST APIs
- JSON payloads
- frontend/backend boundaries
- relational databases
- asynchronous programming
- scalable project structure

## Repository Plan

The repository will grow in phases:

```text
ARGUS/
├── backend/
├── frontend/
├── docs/
├── infra/
└── tests/
```

Each folder will be introduced only when the phase requires it.

## Learning Rule

ARGUS is not just a codebase. It is a guided engineering project. Every folder,
file, endpoint, database table, service, and component should explain:

- what it does
- why it exists
- how it fits into the system
- what alternatives exist
- how industry systems use the same idea
- what mistakes beginners commonly make
